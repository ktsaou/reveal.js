FireHOL

a Linux firewall for humans...

firehol.org
first release 2002, latest release 2017
 
FireHOL is a language to express firewalls.

  • human friendly, even with huge firewalls
  • scriptable, the config is a BASH script
  • supports IPSETs
  • supports SYNPROXY (DDoS protection)
  • creates iptables load balancers
  • creates dynamic firewalls, like auto-blacklisting on portscan, enable service after knock, etc.
  • can load-balance multiple WANs


internet="ppp+"
lan="eth0"
admins="example.firehol.org"

masquerade ${internet}

interface ${lan} home
    policy accept

interface ${internet} world
    policy drop
    protection strong

    server ping accept
    server smtp accept
    server http accept
    server ftp  accept
    server ssh  accept src ${admins}

    client all  accept

router world2home inface ${internet} outface ${lan}
    policy reject
    protection strong

    client all accept

FireQOS

a Linux traffic shapper (and still the only one)

firehol.org
first release 2013, latest release 2017
 
FireQOS is a language to express traffic shapping.

  • human friendly
  • scriptable, the config is a BASH script
  • can shape inbound and outbound traffic
  • supports encapsulation overheads calculation (e.g. ADSL/ATM overheads)
  • supports virtually unlimited nesting of classes, even with link emulation
  • can tcpdump the contents of any class 


interface eth2 world bidirectional rate 100Mbit

    class critical commit 2Mbit
            client dns
            client ntp

    class interactive commit 10Mbit
            match icmp limit 1Mbit

            server ssh
            client ssh
    
    class web_server commit 70%
            server http

    class bulk commit 5Mbit
            server smtp
            client smtp
            server rsync
            client rsync

    class default
            # the rest of the traffic
            # ends up, here

All Cybercrime IP feeds

update-ipsets and iprange

iplists.firehol.org
first release 2015, latest release 5 mins ago (it's a bot)

  • update-ipsets is a bot to download, process and analyze IPSETs.
     
    It downloads IP Feeds from their maintainers, it processes them to extract IPs, monitors the age of the IPs listed, detects their retention policy and checks how they correlate or overlap
     
  • iprange is a high performance ipsets processor (it applies sets theory operations to IPSETs). It is really very fast!
     
  • The site... well it is a monitoring console. update-ipsets generates static JSON files every few minutes. These files are uploaded to the site and are rendered on charts, maps and tables.

So, why people fall in love
with netdata?

netdata dashboards are awesome

  • unbelievably fast
  • highly interactive
  • simple but powerful
  • provide massive insights
  • and unparalleled detail

The background video runs at normal speed,
page scrolling with mouse-wheel,
charts' zooming with SHIFT + mouse-wheel.

netdata dashboards are meaningful

  • metrics are organized into charts
  • charts have a meaning, a scope, a purpose, that is bundled with a title, a description, a unit of measurement and related monitoring alarms
  • charts are organized into families
  • families are organized into apps
  • apps are organized into hosts

More than 30% of the work on netdata, was spent on understanding what each metric monitors, what it means for us (the users) and how it should be presented

netdata is sysadmin friendly

 
 
  • zero disk I/O
  • zero network activity
  • zero configuration
  • zero maintainance
  • zero dedicated resources
     
     
  • 1% CPU of a single core
  • a few MB of RAM
  • almost zero escalated priveleges

 
These depend on the setup details, but for default setups, the above are true

netdata is simple

  1. install it
    netdata knows how to collect more than 5.000 distinct metrics out-of-the-box, all of them with auto-detection, most probably you will only need to configure your other apps to allow netdata access them.
     
  2. optional: configure alarm notifications
    netdata comes pre-configured with more than 100 alarms and alarm templates, that by default sent notifications by email and appear on the dashboard. To use other notification channels, netdata needs to know API keys and recipients.
     
  3. optional: integrate it with your other tools
    netdata can push metrics to Graphite, OpenTSDB, InfluxDB, Prometheus, and all compatible ones, so that other tools like Grafana can use netdata metrics.
     
  4. use it
    to diagnose performance and availability issues

netdata sends alarms

  • alarms are role based
  • each role can send notifications to multiple recipients over multiple channels
  • each recipient on every channel, can filter notifications based on criticality

netdata is extensible

and comes with a large set of plugins

So, what does it take to
run a large FOSS project?

Which projects grow?

  1. do something you love
    you will spend quite some time on it, so you better love it
  2. solve a real problem
    a real need, otherwise it will be a toy
  3. simplify, educate your users
    it should be simple and useful for your users too
  4. make it stunning, awesome
    this contributes a lot to fast growth

How to build a community?

  1. get social
    twitter, facebook, reddit, ... whatever your users use
  2. be responsive
    always respond promptly
  3. release often
    every release is an opportunity to grow the community
  4. be you
    it doesn't matter if you don't know everything - ask your users for help

How to maintain a community?

  1. be open for new ideas
    first understand, then reject
    many great ideas will come from your users
  2. always be polite
    there are many cultures out there
    never get angry, never get ironic
  3. criticism makes the project better
    welcome criticism and learn from it
    try to extract something that will make the project better, ignoring the attitude
  4. always respond and explain
    responsivenes means the project is maintained
    explaining in detail promotes awareness
  5. demand respect for your time
    people have to do their homework before asking for help
    don't use channels that waste your time
